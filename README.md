战争时刻：铁人 (War Moment: Ironman)
 
一款基于 Unity 独立自主开发的 3D 即时战略 (RTS) 游戏 Demo，聚焦于网格建筑放置、单位智能控制与模块化 UI 系统的实现。
 
1. 项目概述 (Project Overview)
 
- 开发引擎：Unity 2022.3.10f1
- 开发语言：C#
- 开发周期：2025年10月 - 2026年3月（独立开发）
- 核心玩法：采集、生存、建造、思考。
- 技术亮点：自定义网格寻路、炮塔与身体解耦的单位AI、基于事件驱动的 UI 架构、自动化资源物流系统。
- 开发目标：实现“3D像素工厂”，结合缺氧、异星工场、纪元1800等经典工厂及自动化类游戏的优点，同时强化战斗设计。
2. 核心功能演示 (Core Features)
 
2.1 精准建筑放置系统
 
- 网格系统：基于  Raycast  与自定义网格算法，实现建筑在地形上的精准吸附；实现细节：拿到地形（terrain）的data数据和transform数据
，在高于地形高度的位置向通过数组遍历提前设定好的网格点发射射线，得到一组地形点高度数据，然后通过二维数组遍历数据并打上遍历记号，
启动BFS，当相邻网格高度一致且相邻网格数量大于10时将其视为“合法网格”，获取组成这些网格的点分别进行取中心点、切分三角形点和设置uv位置
的操作；可视化网格：根据三角形点和uv点的摆放，生成一个空物体并在其上生成mesh，mesh matiaral为unity可视化图形编辑插件所创建的节点化
shader；（具体实现详情请查看GridSystem文件夹中GridDraw脚本）
- 放置系统：通过按键输入选择建筑实现建筑在网格上的放置；吸附功能：基于kd树所开发的查找效率优化算法，可根据二维点字典查找最近的点
实现吸附。按键选择功能：通过两个协程和两个bool值来控制委托回调函数不冲突（具体实现细节请查看GridSystem文件夹中的BuildingPlacer脚本）
采用层级设计，按B键进入第一个协程，游戏暂停，订阅建筑放置函数，此时按b见无法再进入第一个协程，而是直接退出建造并取消暂停。按其他
指定的建筑按键包括B建进入第二个协程，即可开始选择建筑，放置函数中同时订阅两个函数，一个负责鼠标左键放置，一个负责鼠标右键取消建筑
也就是说当进入第一个协程时订阅放置函数，当进入第二个协程时订阅鼠标操作的两个函数，这便是分级设计。（建筑数据脚本请查看
GridSystem文件夹中的BuildingModel_C脚本）[动画](https://github.com/user-attachments/assets/673d2e85-73aa-41b7-a5bc-84377952c214)
 
2.2 高级单位控制逻辑
 
- 组件解耦：实现士兵身体移动与炮塔索敌攻击的完全解耦，支持炮塔独立转向、自动归位。
- 目标切换：支持玩家手动指定攻击目标，目标死亡后自动切换至最优目标。
- 协程管理：使用  Coroutine  优雅处理攻击冷却、移动插值等异步逻辑，确保性能稳定。
 
2.3 模块化 UI 框架
 
- 事件驱动：基于 C#  Delegate  与  Event  实现 UI 与逻辑层的彻底解耦。
- 状态管理：封装 UI 面板（如 Loading、MainMenu、BattleHUD）的显示、隐藏与传参逻辑。
 
2.4 资源与自动化物流
 
- 模拟 RTS 经典的资源采集与运输逻辑，实现单位自动寻路、资源自动交付。
 
3. 技术架构 (Technical Architecture)
 
3.1 设计模式应用
 
- 单例模式 (Singleton)：用于管理 GameManager、UIManager 等全局核心模块。
- 事件中心 (Event Center)：统一管理游戏内所有事件（如“资源变更”、“单位死亡”），降低脚本耦合度。
- 工厂模式：用于士兵与建筑的批量创建与回收，优化内存使用。
 
3.2 关键技术点
 
模块 核心技术 解决的问题 
物理与检测  Raycast ,  LayerMask ,  OverlapBox  精准的鼠标拾取与地形交互，避免建筑重叠 
AI 与寻路 A* 寻路算法（自定义）+ Unity NavMesh 实现单位的高效路径规划与避障 
动画与状态 Animator Controller + 状态机 平滑过渡单位的移动、攻击、死亡状态 
数据持久化 JSON + PlayerPrefs 保存玩家设置与游戏进度 
 
4. 开发环境 (Development Environment)
 
- IDE：Visual Studio 2022
- 版本控制：Git + GitHub
- 建模工具：Blender（用于简易模型与动画制作）
- 第三方插件：Doozy UI（UI 管理）、Odin Inspector（开发效率）
 
5. 快速开始 (Quick Start)
 
1. 克隆本仓库： git clone https://github.com/afhhsdasd/Unity-.git 
2. 使用 Unity Hub 打开项目（确保版本为 2022.3.10f1）。
3. 打开  Scenes/Map_坠落地.unity  场景，点击运行即可开始游戏。
 
6. 操作说明 (Controls)
 
- 键盘wdtc：选择建筑、确认放置。
- 鼠标右键：移动单位、下达攻击指令。
- 数字键：快速切换建造面板。
- B：暂停游戏。
- ESC：返回主菜单
7. 个人开发总结 (Developer Notes)
 
本项目是我对 RTS 游戏开发的一次深度探索。在开发过程中，我重点攻克了复杂单位的组件化设计与大型项目的代码可维护性问题。通过封装事件中心和采用单例模式，我构建了一个易于扩展的架构，为后续功能迭代打下了坚实基础。同时，在性能优化方面，我通过对象池技术减少了频繁实例化带来的卡顿，确保了游戏的流畅运行。
